Fixed Dynamic Media Update System

Issue Detected

We found that the website is using a flat file structure for media files, while our dynamic system was designed for a subfolder organization. This is why your updates weren't showing up.

Solution Implemented

We've updated the media manager script to handle both the flat legacy structure and the new subfolder organization. Here's how to properly update images now:

How to Update the Services Page Image

To update the "Wedding planner with couple in mountains" image on the services page:

  1. Identify the correct file path:
    The image is referenced as assets/images/service-intro.jpg in the HTML
  2. Replace the file directly:
    Save your new image as service-intro.jpg and place it in src/assets/images/ (not in a subfolder)
  3. Refresh the page:
    Open the services page and refresh to see your changes

Other Common Images to Update

Homepage:

  • Hero Image or Video: src/assets/images/hero-poster.jpg and src/assets/videos/hero.mp4
  • Featured Images: src/assets/images/featured-1.jpg, src/assets/images/featured-2.jpg, etc.
  • Testimonial Images: src/assets/images/testimonial-1.jpg, src/assets/images/testimonial-2.jpg, etc.

Services Page:

  • Intro Image: src/assets/images/service-intro.jpg
  • Case Study Images: src/assets/images/case-before-1.jpg, src/assets/images/case-after-1.jpg, etc.

Venues Page:

  • Intro Image: src/assets/images/venues-intro.jpg
  • Region Images: src/assets/images/venues/banff-region.jpg, src/assets/images/venues/canmore-region.jpg, etc.
  • Venue Detail Images: src/assets/images/venues/banff-springs-main.jpg, src/assets/images/venues/banff-springs-1.jpg, etc.

Monitoring Updates

All image updates should now be properly tracked in the src/assets/media-updates.log file.

If you're not seeing updates in this file, you can manually check its contents to see what's being recorded.
